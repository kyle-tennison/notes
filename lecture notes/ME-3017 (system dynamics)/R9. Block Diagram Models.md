# R9. Block Diagram Models

## Block Diagrams (9.5)

### Block Diagram Symbols

1. Arrow: Self explanatory, shows cause/effect
2. Block: Represents the input/output relation of the transfer function
3. Circle: The *summer*, which represents addition or subtraction, depending on the accompanying
4. "Takeoff Point:" A probe of a value from a certain part of the diagram. Doesn't change what it's probing.

### Series Elements and Feedback Loops


![[IMG_1795 2.jpeg]]

- (a) Series Block
- (b) Series Block Simplified
- (c) Feedback Loop
- (d) Feedback Loop Simplified

For **series** blocks, it is pretty self explanatory; for instance, in (a):

$$B(s) = F(S)T_1(s)$$

and then

$$X(s) = B(s)T_2(s) = F(s)T_1(s)T_2(s)$$

they just string together. Should be easy to see how they can be simplified like (b).

For **feedback loops**, we need to consider the output at the time we define the input. This is what confused me at first. When defining $A(s)$, we need to consider how $B(s)$, even through $B(s)$ depends on $X(s)$ which depends on $A(s)$. It might seem like a circular dependency, but it can be solved. For (c) above, we get the system:

$$\begin{cases}
A(s) = F(s)-B(s)\\\\
B(s)= H(s)X(s)\\\\
X(s)=G(s)A(s)
\end{cases}$$

Solving for $X(s)$ gives:

$$X(s) = \frac{G(s)}{1+G(s)H(s)}F(s)$$
This is where the equation for (d) comes from which lets us simplify any feedback loop.
